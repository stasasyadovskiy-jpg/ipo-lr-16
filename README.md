## Общий сценарий работы (Workflow)

Для всех вариантов алгоритм работы приложения одинаковый:

1. **Ввод:** Пользователь заходит на главную страницу (/), где видит форму с двумя элементами:
    - Поле для загрузки файла (.xlsx).
    - Дополнительное поле ввода (число или текст) — *параметр для расчета* (зависит от варианта).
2. **Обработка:** Сервер принимает файл, читает его через pandas, производит вычисления с учетом введенного параметра.
3. **Вывод:** Браузер автоматически предлагает скачать готовый файл (report.xlsx).

---

## Технические требования

- **Стек:** Python, Flask, Pandas, OpenPyxl (или XlsxWriter).
- **Frontend:** Простая HTML-форма.
- **Backend:**
    - Маршрут GET / — отдает HTML-форму.
    - Маршрут POST /process — принимает файл, обрабатывает и возвращает результат через send_file.
    ### Вариант 2: Конвертер валют для прайс-листа

**Сценарий:** Менеджер получает прайс-лист от поставщика в долларах и хочет получить прайс в рублях по текущему курсу.

- **Входной файл:**
    - Товар (строка)
    - Цена в USD (число)
    - Наличие (число)
- **Параметр в форме:** Текущий курс USD (число, например, 92.50).
- **Задание на обработку:**
    1. Создать столбец Цена в RUB = Цена в USD * Курс.
    2. Если Наличие равно 0, в столбце Цена в RUB вместо цены написать текст "Нет в наличии".
    3. Отсортировать товары: сначала дорогие, потом дешевые.
- **Итоговый файл:** Обновленный прайс-лист.
